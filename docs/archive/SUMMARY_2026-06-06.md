### Bug #1: 奶穿规则未实现 ⭐️⭐️⭐️⭐️⭐️

**严重性**: Critical  
**位置**: `backend/engine/game.py:861-904` `_night_resolve()`  
**违反规则**: CLAUDE.md "同守同救（奶穿）" 规则

**问题描述**:
```python
# 旧代码（错误）
if (
    wolf_target_id
    and not self.state.night_actions.witch_save          # 女巫没救
    and wolf_target_id != self.state.night_actions.guard_target_id  # 守卫没守
):
    deaths.append({"player_id": wolf_target_id, "reason": "wolf"})
```

**规则要求**:
> 若同一玩家同时被守卫守护和女巫解药救，该玩家**仍死亡**。

**当前行为**: 守卫守 **或** 女巫救 → 存活（错误）  
**正确行为**: 守卫守 **且** 女巫救 → 死亡（奶穿）

**修复代码**:
```python
# 新代码（正确）
wolf_target_id = self.state.night_actions.wolf_target_id
guard_target_id = self.state.night_actions.guard_target_id
witch_save = self.state.night_actions.witch_save

# 奶穿判定：同时被守卫守护和女巫解药救 → 死亡（CLAUDE.md 核心规则）
both_guarded_and_saved = (
    wolf_target_id
    and witch_save
    and wolf_target_id == guard_target_id
)

if wolf_target_id:
    if both_guarded_and_saved:
        # 同守同救（奶穿）→ 死
        deaths.append({"player_id": wolf_target_id, "reason": "milk_through"})
    elif not witch_save and wolf_target_id != guard_target_id:
        # 既没被守也没被救 → 死
        deaths.append({"player_id": wolf_target_id, "reason": "wolf"})
    # else: 只被守或只被救（但不同时）→ 活
```

**测试场景**:
| 场景 | 狼刀 | 守卫 | 女巫 | 修复前 | 修复后 | 正确 |
|------|------|------|------|--------|--------|------|
| 只守卫守 | P1 | P1 | 否 | 活 | 活 | ✅ |
| 只女巫救 | P1 | P2 | 是 | 活 | 活 | ✅ |
| 同守同救 | P1 | P1 | 是 | **活** | **死** | ✅ |
| 都没保护 | P1 | P2 | 否 | 死 | 死 | ✅ |

**影响分析**:
- 影响范围: 100% 的游戏
- 规则违反: 严重
- 游戏平衡: 可能让玩家永生

### Bug #2: 狼人可以刀狼队友 ⭐️⭐️⭐️⭐️⭐️

**严重性**: Critical  
**位置**: `backend/engine/actions.py:52-59` `ActionValidator.validate()`  
**违反规则**: CLAUDE.md 关键规则 #1 "狼刀目标不能是狼人"

**问题描述**:
```python
# 旧代码（错误）
if target.id == actor.id and decision.action_type in {
    ActionType.VOTE,
    ActionType.ATTACK,    # 只检查了不能刀【自己】
    ActionType.DIVINE,
    ActionType.WITCH_POISON,
    ActionType.SHOOT,
}:
    return False
```

**规则要求**:
> 狼刀目标不能是狼人：狼人击杀的候选目标只能是**非狼人存活玩家**。狼人不能选择自己或狼队友作为击杀目标。

**当前行为**: 只检查 `target != self`，狼人可以刀队友  
**正确行为**: 检查 `target.alignment != WOLF`，禁止刀队友

**修复代码**:
```python
# 新代码（正确）
if target.id == actor.id and decision.action_type in {
    ActionType.VOTE,
    ActionType.ATTACK,
    ActionType.DIVINE,
    ActionType.WITCH_POISON,
    ActionType.SHOOT,
}:
    return False

# 狼人不能攻击狼队友（CLAUDE.md 关键规则 #1）
if decision.action_type == ActionType.ATTACK:
    if target.alignment == Alignment.WOLF:
        return False
```

**测试场景**:
| 场景 | 攻击者 | 目标 | 修复前 | 修复后 | 正确 |
|------|--------|------|--------|--------|------|
| 狼刀自己 | P1(狼) | P1(狼) | ❌ 拒绝 | ❌ 拒绝 | ✅ |
| 狼刀队友 | P1(狼) | P2(狼) | **✅ 通过** | **❌ 拒绝** | ✅ |
| 狼刀好人 | P1(狼) | P3(民) | ✅ 通过 | ✅ 通过 | ✅ |

**影响分析**:
- 影响范围: 100% 的狼人局
- LLM Agent 可能误判刀队友
- `_majority_target()` 不做过滤，会直接采纳错误目标

### Bug #3: 屠边判定缺失 ⭐️⭐️⭐️⭐️

**严重性**: High  
**位置**: `backend/engine/game.py:1581-1599` `_check_win()`  
**违反规则**: CLAUDE.md 胜利条件 "狼人赢（屠边）"

**问题描述**:
```python
# 旧代码（不完整）
if not alive_wolves:
    winner = Alignment.VILLAGE  # ✅ 正确
elif len(alive_wolves) >= len(alive_village):
    winner = Alignment.WOLF  # ⚠️ 只检查了人数平局（屠城）
```

**规则要求**:
> - 好人赢：所有狼人出局
> - 狼人赢（**屠边**）：所有神民出局 **或** 所有村民出局
> - 狼人赢（屠城）：所有好人出局

**当前行为**: 只检查人数平局（屠城），未检查屠边  
**正确行为**: 同时检查屠边（所有神死 或 所有村民死）

**修复代码**:
```python
# 新代码（完整）
# 好人赢：所有狼死
if not alive_wolves:
    winner = Alignment.VILLAGE
    reason = "all_wolves_dead"
# 狼人赢：屠城（人数平局或狼人更多）
elif len(alive_wolves) >= len(alive_village):
    winner = Alignment.WOLF
    reason = "wolves_reached_parity"
# 狼人赢：屠边（所有神死 或 所有村民死）
else:
    gods = [p for p in alive_village if p.role in {Role.SEER, Role.WITCH, Role.HUNTER, Role.GUARD, Role.IDIOT}]
    villagers = [p for p in alive_village if p.role == Role.VILLAGER]
    
    if not gods and villagers:
        # 所有神出局，只剩村民 → 狼人屠边胜利
        winner = Alignment.WOLF
        reason = "all_gods_dead"
    elif not villagers and gods:
        # 所有村民出局，只剩神 → 狼人屠边胜利
        winner = Alignment.WOLF
        reason = "all_villagers_dead"
```

**测试场景**:
| 场景 | 存活 | 修复前 | 修复后 | 正确 |
|------|------|--------|--------|------|
| 所有狼死 | 5 好人 | 好人赢 | 好人赢 | ✅ |
| 人数平局 | 2狼+2好 | 狼赢 | 狼赢 | ✅ |
| 所有神死 | 2狼+3村+0神 | **继续** | **狼赢** | ✅ |
| 所有村民死 | 2狼+0村+3神 | **继续** | **狼赢** | ✅ |
| 神民都有 | 2狼+2村+3神 | 继续 | 继续 | ✅ |

**影响分析**:
- 影响范围: 部分游戏（狼人优势但未达平局时）
- 游戏可能拖得过长
- 狼人应该提前胜利但未判定
