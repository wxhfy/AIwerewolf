# 后端验收标准

| 模块 | 验收条件 | 失败条件 | 对应命令 | 输出文件 |
|------|---------|---------|---------|---------|
| **DB** | PostgreSQL 可连接，20 张表存在（games/players/game_events/agent_decisions/game_snapshots/votes/evaluations/agent_versions/leaderboard_entries/review_reports/published_reviews/strategy_knowledge_docs/strategy_graph_links/role_strategy_cards/persona_role_adapters/strategy_patches/evolution_tournaments/knowledge_usage_feedback/evolution_rounds/personas） | 连接失败或表缺失 | `preflight.check_db()` | - |
| **LLM** | 客户端可用，health check JSON 返回 `ok`，同步调用 `chat_sync()` 返回合法响应 | client unavailable 或 health check 失败，或响应为空 | `create_client().chat_sync()` | - |
| **LLM 降级链** | 主模型可用时使用主模型；主模型 3 次失败后自动降级到备选模型；备选模型也失败时降级到 heuristic | 主模型不可用时对局直接崩溃，无降级逻辑 | Game Health Report 中的 `agent_downgrade_count` | `game_health_report.json` |
| **Game Engine** | 标准 7 人局全流程跑通（NIGHT_START → GAME_END），所有阶段按规则顺序执行，无跳过或死循环；12 人局同样跑通 | 阶段卡死、对局无法完成、技能结算错误（如女巫双药重复使用） | `run_game_experiment.py` | `game.log` |
| **Agent Decision** | 每个 Agent 在每个阶段产出合法决策（发言非空、投票目标为存活玩家、技能目标合法）；决策写入 `agent_decisions` 表；parsed_action 包含 `_auto_injected_strategies` 和 `_tool_trace` | Agent 无响应超时后无降级；产出非法决策（如投票给已死亡玩家）；decision 记录缺失关键字段 | `test_cognitive_agent.py` / `test_cognitive_integration.py` | `agent_decisions.jsonl` |
| **Information Isolation** | 12 项边界检查全部通过：非狼阵营看不到 `known_wolves`；非预言家看不到查验结果；非女巫看不到用药信息；村民看不到任何 private_events；每个 Agent 的 observation 不含其他角色的私密信息 | 任意一项信息泄露（如村民的 PlayerView 中出现"3号是狼人"） | `visibility.py` 的 `for_player()` 方法 + 手动审计 | `isolation_check_result.json` |
| **Strategy Retrieval** | BM25 检索在 500ms 内返回结果；检索关键词与命中文档的相关性可验证（人工抽样 Top-3 命中）；active/candidate/deprecated 三层隔离，Agent 只能检索 active 文档 | 检索超时 > 2s；active 池不包含相关策略时返回空但不崩溃；candidate/deprecated 文档泄露到检索结果中 | `eval_retrieval.py` / `eval_final_retrieval.py` | `retrieval_eval_results.json` |
| **Anti-Pattern Injection** | 当开启 anti_pattern 时，角色策略卡的 anti_pattern 层正确注入到 LLM prompt 中；反模式规则可被 Agent 在决策中引用（从 `auto_injected_strategies` 中可验证） | anti_pattern 未注入或被忽略；反模式文本被截断（total prompt > max_tokens） | `test_cognitive_integration.py` | `anti_pattern_injection_verify.json` |
| **Knowledge Snapshot** | 实验前可创建 strategy_snapshot，记录当前 active docs 的 id + content_hash；实验期间即使有新 lesson 入库也不影响 snapshot 环境 | snapshot 创建失败或内容为空；实验期间 active 池被新 lesson 污染 | `manage_knowledge.py snapshot create` | `snapshot.json` |
| **Track B Scoring (PerStepScorer)** | Tier 1 对所有决策产出初始 correctness 分（100% 覆盖率）；Tier 2 对 correctness ∈ [0.3, 0.7] 的决策自动触发单 Judge LLM 评分；Tier 3 对满足条件的决策触发三法官 panel；每个 ScoredStep 包含 is_mistake/is_highlight 标记 | 评分缺失（某决策无分）；Tier 分级逻辑错误（high-conf 进入 Tier 3）；is_mistake 标记与实际人工判断偏离 > 30% | `per_step_scorer.py` | `scored_steps.json` |
| **Track B Review (PlayerReviewReport)** | 每个玩家产出一份 PlayerReviewReport，含得分汇总（平均分/最低分/失误数/高光数）、失误列表（含 step_id + mistake_type）、改进建议；全局 ReviewReport 含阵营对比和阶段分布 | 玩家 report 缺失；report 中 step_id 与 agent_decisions 无法对应；改进建议为空 | `generate_final_review.py` | `review_report.json` |
| **Track C Knowledge (KnowledgeAbstractor)** | 每局赛后从 ScoredStep 中提取 lessons（含 situation_pattern / target_role / recommended_action / confidence_score）；lessons 写入 `strategy_knowledge_docs` 表（status=candidate，source_game_id 非空）；默认不晋升到 active | 赛后未产出任何 lesson；lesson 的 source 信息丢失；lesson 错误地自动晋升到 active | `knowledge_abstractor.py` | `abstracted_lessons.jsonl` |
| **Track C Evolution Loop** | candidate lesson 可通过规则阈值或人工审核晋升到 active；晋升后在新对局中可被检索到（从下一局的 tool_trace 中验证）；非跨轮 knowledge 泄露（上一局的 lesson 不应直接影响本局评分） | 晋升逻辑失效；active 池被低质量 lesson 污染；跨轮 knowledge 循环导致评分虚高 | `manage_knowledge.py promote` + `evolution_test.py` | `evolution_loop_verify.json` |
| **Experiment (多 Tier 对比)** | 4 个 Tier 至少各 30 局完成；跨 Tier 同 index seed 保证 MBTI 分配公平；输出 Bootstrap 95% CI + Permutation test p-value + Cohen's d | 任何 Tier 未达到 30 局最小样本量；seed 映射错误导致某角色在某个 tier 人格分布不均 | `multi_tier_experiment.py` | `experiment_results.json` |
| **Experiment Leaderboard** | 按胜率 + 过程分 + 策略使用率三维排名，Tier 间可区分；至少 one tier 在至少 one metric 上显著优于 baseline（p < 0.05） | 所有 tier 无显著差异；排名与预期方向相反；数据无法复现 | `multi_tier_experiment.py` + `build_minimal_leaderboard.py` | `leaderboard.json` |
| **Report Export** | 支持导出结构化 JSON 报告（PlayerReviewReport + ScoredSteps + AbstractedLessons）；支持导出 HTML 报告（单局 + 全局 dashboard）；支持证据链 JSON（Observation → Decision → Score → Lesson 完整链路） | 报告导出失败或数据不完整；HTML 渲染错误（JS Console 报错）；证据链断开 | `render_dashboard_html.py` / `build_single_game_report_data.py` | `dashboard.html` / `game_report.json` |
| **Preflight / Health Check** | 启动前运行 `preflight.check_all()`：DB 连接 + 表存在 + LLM 可用 + 策略库 active 数量 >= 阈值 + 规则配置文件可解析；所有检查项通过 | 任意一项预检失败 | `preflight.check_all()` | `preflight.log` |
| **Error Handling / Resilience** | LLM 调用 3 次重试（指数退避 1s/2s/4s）；DB 连接断开自动重连；单局崩溃不影响同 tier 其他局；所有异常写入 structured error log | 未捕获异常导致进程退出；error log 丢失；重试逻辑失效 | Game Health Report 中的 `error_count` / `crash_count` | `error.log` |
| **Configuration Validity** | `rule_variant_standard.yaml` 语法正确、角色列表与 RoleRegistry 一致、阶段顺序符合狼人杀规则；`strategy_library.yaml` 中的策略文档每条的 required fields 非空 | 配置文件解析失败；角色名不存在于 RoleRegistry；字段缺失导致运行时 panic | `config.py` 在初始化时自动校验 | `config_validation.log` |
| **Concurrency / Multi-Game** | 同时运行 4 局不冲突（独立 game_id + 独立 agent_decisions + 独立 DB 连接）；多进程写入同一 PostgreSQL 数据库无竞态条件 | 竞态导致数据写错行、game_id 冲突、DB 死锁 | `multi_tier_experiment.py` 4 tier 并行 | 所有 game 的决策记录完整且无交叉污染 |
