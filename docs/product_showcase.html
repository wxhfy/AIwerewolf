01

项目定位

一个把「玩游戏」「评游戏」「学游戏」三件事打通的多智能体系统——既是可观战的 AI 狼人杀产品，也是研究"分层认知 + 知识自进化是否真能让 Agent 变强"的实验平台。

主线含义对应能力
对战 Play多 Agent 在完整规则下博弈游戏引擎 + 认知 Agent + 实时观战
复盘 Evaluate量化评估每个玩家每一步的质量LLM Judge 评审团 + 多维打分 + 验收门
进化 Evolve把复盘结论变成更强的策略知识提炼 → 知识库 → Agent 版本 → 晋级

02

设计哲学

后面所有模块的取舍，都能追溯到这四条贯穿始终的原则。

信息隔离优先

每个 Agent 只能看到角色被允许的信息。狼人杀的全部乐趣与研究价值都建立在"信息不对称"上；一旦泄漏上帝视角，博弈就退化成作弊。

严格无兜底

追求每一步都是真实 LLM 决策、每一步都触发知识检索。只有"不掺假"的对局，才能用来验证模型到底强不强、知识到底有没有用。

证据可追溯

从最终入库的知识能一路反查到原始那一刻的决策与事件。研究结论必须可复现、可审计，不能是黑箱里冒出来的分数。

参考但不照搬

系统研究了 10 个开源实现，理解设计后自己重写核心。既少踩坑，又保证核心可控、可改、无许可证风险。

Layer 1 · 怎么思考

MBTI 人格

认知风格、说话语气；32 个命名角色的人格系统。

Layer 2 · 我是谁

Role 身份

角色技能、胜利条件、反模式清单。

Layer 3 · 怎么赢

策略知识

运行时从历史复盘知识库动态检索经验，注入决策——这一层是"自进化"的落点。

04

核心模块设计：动机 · 为什么 · 好处

不堆代码，重点讲每个核心模块当初是怎么设计出来的、为什么这么选、带来了什么好处。

游戏引擎 可信的规则底座

管回合流转、行动校验、夜晚结算、胜负判定与信息隔离，正确性零容忍。
设计动机
综合 wolfcha 的产品级阶段模型、open_mafia 的事件驱动、werewolf-brain 的夜晚序列，取各家之长后自研重写。
为什么这么设计
把游戏建成阶段状态机而非顺序脚本——狼人杀有大量分支（平票 PK、猎人开枪、自爆）；夜晚行动"视为同时发生"——贴合线下真实裁判规则。
好处
加新角色/规则往往只是新增一个阶段或一条转换，改动面小、可扩展；对局结果符合常识，复盘时不会出现"规则错误导致的伪失误"。

认知 Agent Observe → Think → Act

每个 AI 玩家的"大脑"，承载项目 70% 的能力评分。
设计动机
经典 AIWolf 接口把"思考"当黑箱；我们要一个思考过程本身就结构化、可观测、可复盘的 Agent，于是重构成显式的"观察→思考→行动→反思"循环。
为什么这么设计
给 Agent 配信念 + 量化人格 + 记忆 + 社会模型的内部状态，而非每次裸调 LLM（避免金鱼记忆、人格漂移）；思考阶段用有限轮次的工具调用按需检索，而非把海量知识无差别塞进上下文。
好处
发言更连贯、人格更鲜明、决策可解释；每条决策都带完整工具调用轨迹，能精确还原"它用了哪条知识"——这是验证自进化的基础。

评测体系 Track B 量化每一步

客观回答"这一步打得好还是烂"，并生成结构化复盘。
设计动机
参考 WereWolfPlus 的多模型评测，但碰到现实矛盾：LLM 当裁判最准，却又慢又贵，全用评审团做不了规模实验。
为什么这么设计
按"影响力 × 模糊度"做三级打分级联：~85% 规则判定（零成本）、~12% 轻量 LLM、~3% 评审团（3 评委 + Critic + 截尾均值）。贵的算力只花在"既关键又难判"的决策上。
好处
在可接受成本下拿到接近"全评审团"的质量；实测对赢家/输家的区分达大效应量（见设计历程阶段 4）。

知识自进化 Track C 让经验回流

把复盘结论提炼成可复用知识，回流给下一代 Agent。
设计动机
希望对局经验像人一样"沉淀成手感"，而不是看完就过去。
为什么这么设计
知识有生命周期（candidate→active→deprecated）和 L0–L4 置信分级，会自我净化；注入前过4 重安全过滤，其中专门有一条堵死"赛后信息泄漏回当前对局"——这是研究有效性的命门。
好处
享受经验回流红利的同时严守信息隔离底线（活跃池零污染）；能用"知识归因"反查哪条知识真被用上、且帮到了赢面。

LLM 接入层

为什么
一套接口适配 Anthropic / OpenAI 双格式，提供商 `.env` 可切；另备一个 fake 假模型。
好处
换模型、做多模型对比时业务零改动；用假模型可离线、可重复跑 354 个测试。

实时通信与前端

为什么
引擎是同步逻辑，前端要实时，用观察者 + 快照队列解耦；前端把"游戏真实状态"与"观众展示状态"分离（打字机、昼夜眨眼转场）。
好处
观感接近直播、天然支持重连；非技术读者也能看懂系统在干嘛。
