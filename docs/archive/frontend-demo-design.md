# 前端 Demo 设计说明

## 一句话原则

普通观众看「这局发生了什么」。
全局视角看「系统为什么这么决策」。

前端不应让用户在同一个页面里同时看到直播、调试器、后台管理和论文实验。Demo 可以用一个按钮切换视角，但每个视角的内容边界必须清楚。

## 页面分工

### 大厅页 `/`

目标：创建一局可演示的对局，不暴露高级参数。

应该显示：
- 对局模式：AI 对战 / 真人参与
- 玩家人数
- 真人参与时的座位选择
- 开始按钮
- 顶部入口：角色库、进化看板、设置

不应该显示：
- Seed
- API Key
- 模型 Provider / Model / Base URL
- 夜间行动、决策记录、调试信息

### 设置弹窗

目标：收纳 Demo 配置和高级参数，避免污染大厅主流程。

应该显示：
- 默认视角：普通观众 / 全局视角
- 语言
- Seed：用于复现同一局配置
- 模型调用配置：Provider、模型名称、Base URL、API Key

约束：
- API Key 只保存在浏览器本地设置中，不在对局页面展示。
- 当前阶段不改后端，因此模型调用配置只是前端配置入口；真正接入后端前，不把真实 Key 写进代码、文档或 commit。

### 对局页 `/room/[id]/play`

目标：让非技术用户看懂一局狼人杀，也让 Demo 讲解者能切到全局视角解释系统决策。

普通观众应该显示：
- 当前天数 / 阶段
- 玩家姓名、座位、存活状态
- 公开发言
- 公开投票过程与投票结果
- 死亡 / 遗言 / 游戏结束结果
- 夜晚统一状态文案，例如「夜晚行动中，隐藏身份与夜间目标」

普通观众不应该显示：
- 隐藏身份、阵营、夜间角色高亮
- 狼队刀人目标、守卫守护目标、预言家查验目标、女巫用药目标
- AI 决策记录、reasoning、全局 ActionPanel
- 狼队夜间商议聚合信息

全局视角应该显示：
- 普通观众的全部公开内容
- 玩家身份与阵营
- 夜间角色行动高亮
- 夜间行动聚合结果
- 关键决策记录与可解释信息
- AI ActionPanel / 调试性控制入口

全局视角不应该显示：
- 后台管理配置
- 实验指标看板
- API Key 明文
- 与当前对局无关的论文式统计解释

### 评估看板 `/eval/dashboard`

目标：展示 Track B/C 指标，不混入观战主页面。

应该显示：
- 评分区分度图表
- 进化轮次趋势
- Patch 状态
- 验收门通过率

不应该显示：
- 单局观战时间线
- 玩家操作面板
- API Key / Seed 配置

## 风格规范

- 使用统一主题 token：`bg-background`、`bg-cardBackground`、`border-border`、`text-textPrimary`、`text-text-sub`、`text-primary`。
- 页面级布局保持克制、信息密集但清晰；不要把运营页、实验页和观战页混在一个界面。
- 卡片只用于重复项、弹窗和工具面板；主页面区域不再额外套大卡片。
- 普通观众与全局视角的切换是 Demo 辅助能力，不是权限系统；真实权限隔离仍以后端 Visibility 为准。
